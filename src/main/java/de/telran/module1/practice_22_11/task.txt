Техническое задание
Название проекта:
Управление музыкальной коллекцией

Цель проекта:
Разработать приложение для управления музыкальной коллекцией,
которое позволит пользователям создавать, редактировать и оценивать
песни, а также организовывать музыку в плейлисты.

Основные требования к приложению:
1. Классы и Enum:
1.1. Создать Enum класс Rating, который представляет оценки для песен (1-5 звезд).

1.2. Создать Enum класс Genre, представляющий жанры музыки.

1.3. Создать Enum класс Playlist, представляющий плейлисты,
с возможностью задания имени и хранения списка песен.

1.4. Создать класс Song, представляющий песню, с полями:

Название (String title)
Исполнитель (String artist)
Жанр (Genre genre)
Рейтинг (Rating rating) с возможностью установки и получения.
Статическим полем int songCount для подсчета созданных песен.
Переопределенным методом toString() для удобного вывода информации о песне.
1.5. Создать класс Album, представляющий альбом, с полями:

Название альбома (String title)
Исполнитель альбома (Artist artist)
Список песен (List<Song> songs) в альбоме.
1.6. Создать класс Artist, представляющий исполнителя, с полями:

Имя исполнителя (String name)
Список альбомов (List<Album> albums) исполнителя.
1.7. Создать интерфейс PlaylistsManager со следующими методами:

void createPlaylist(String name): Создание нового плейлиста с заданным именем.
void deletePlaylist(String name): Удаление плейлиста по имени.
List<Playlist> getAllPlaylists(): Получение списка всех плейлистов.
1.8. Создать класс User, представляющий пользователя приложения, с полями:

Имя пользователя (String username)
Список плейлистов (List<Playlist> playlists) пользователя.
Реализацией интерфейса PlaylistsManager.
Методами для добавления (addToPlaylist) и удаления (removeFromPlaylist) песен в плейлисте.
Методом rateSong для оценки песни.
2. Функциональность:
2.1. Пользователь может создавать, редактировать и удалять плейлисты.

2.2. Пользователь может добавлять и удалять песни из плейлистов.

2.3. Пользователь может оценивать песни на шкале от 1 до 5 звезд.

2.4. Пользователь может просматривать информацию о песне,
включая ее название, исполнителя, жанр и рейтинг.
